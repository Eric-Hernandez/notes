CSS Units of Measurement

Choices, choices...

What units of measurement do we use when specifying box model, width, height, etc. properties? We utilize screen units, including:

  • pixels (px)
  • percentage (%)
  • ems (em)
  • rems (rem)

Reminder: The Web is Not Print

When you design for print, you use print units:

  • points
  • picas
  • inches
  • centimeters
  • millimeters

DO NOT use print units when designing for digital media. Use screen units.

One exception, which we'll get to: Print style sheets.

Pixels

Pros

  • allows for very precise control of design
  • what you see is what you get (WYSIWYG)

Cons

  • poor choice for fluid, responsive design
  • not ideal for re-sizing text, designs

Syntax

font-size: 30px;

Relative Units

Percentage, ems and rems are relative units.

This means their values are initially based on a very high-level measurement (usually in pixels) set at the browser, html or body level.

Percentage

Pros

  • HTML defaults are calulated expressed in percentage
  • great for fluid, responsive design

Cons

  • requires calculation based on parent font size
  • very precise caluclations can disrupt layout

Syntax

width: 90%;

Ems

Pros

  • great for fluid, responsive design
  • once a base font is set, everything can be relative (to the parent)

Cons

  • requires calculation based on parent font size
  • very precise caluclations can disrupt layout
  • inherits a parent's property

Syntax

margin: 2em;

Root Ems

Pros

  • based on the html element's font size
  • a little more predictable than percentage or ems
  • great for fluid, responsive design

Cons

  • requires calculation
  • can still result in precice, layout-disrupting calculations
  • older browsers (IE 8 or less, Opera Mini) lack support

Syntax

font-size: 1.25rem;
