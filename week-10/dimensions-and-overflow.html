Dimensions & Overflow

Sizing Boxes

As we have seen with the block elements and the box model, content will take up whatever room we give it... unless we specify otherwise.

When we want to create layouts, we can apply constraints:

  • width
  • height
  • overflow

Dimensions

Width is often helpful for layout and design since we are often limited (horizontally) by screen size.

We can specify height but it really is not necessary since scrolling is an accepted practice when using a web interface.

  • width
  • max-width
  • min-width
  • height
  • max-height
  • min-height

Overflow

What happens if we have content which is greater in size than the box we've provided for it?

We can specify overflow properties, which handle how that content is displayed within its parent element.

visible
content breaks the box's dimensions
hidden
hides content which would normally extend beyond the box's dimensions
scroll
adds vertical and horizontal scroll bars
auto
adds vertical and/or horizontal scroll bars if necessary
